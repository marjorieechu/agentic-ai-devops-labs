# CrewAI

CrewAI is an orchestration framework for building collaborative AI agents and structured workflows.

According to the current CrewAI documentation, the platform combines:

- `Crews` for teams of autonomous agents
- `Flows` for structured, event-driven workflow control

## Why It Matters

CrewAI is useful when you want both autonomy and control.

That makes it a good fit for:

- multi-agent task execution
- long-running workflows
- operational automations
- systems that need state, branching, and tool use

## Core Concepts

### Crews

Crews are teams of role-based agents that collaborate on delegated tasks.

### Flows

Flows define the application logic around those agents, including:

- state management
- event-driven steps
- branching and control flow

## When I Would Use It

- when a project needs multiple specialized agents
- when I want more structure than simple agent chat loops
- when I need production-minded workflow control

## Strong Use Cases

- incident response automation
- research and reporting systems
- support triage workflows
- DevOps assistants with role-based agents

## Learning Checklist

1. Build a simple crew with two specialized agents
2. Add tools to one or more agents
3. Wrap the crew inside a flow
4. Add logging and structured outputs
5. Compare the same use case against LangGraph
